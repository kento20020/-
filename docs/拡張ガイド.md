# 拡張ガイド — 要素の足し方（アップデートを簡単にする手引き）

このゲームは **「データ(`web/data.json`)＋レジストリ関数」** で要素を足せる構造。
多くの追加は **engine.ts を編集せず** に済む。各手順の最後に必ず：

```bash
cd web && npm run check     # doctor＋テスト＋KPIアサート＋ビルド
```

- **見た目/数値だけの追加（挙動は同じ系統）** → ゴールデンは緑のまま（再生成不要）。
- **バランス（数値）を変えた** → `npm run gen-fixtures` で基準を更新し、`docs/KPIレポート_1000runs.txt` と
  `docs/統計レポート_分析.md` を更新（`CLAUDE.md` のバランス変更プロトコル参照）。

---

## 武器を足す
1. `web/data.json` の `weapons[]` に1件追加：`{ id, name, atk, defMod, bonusPoison, note }`。
2. **効果付き武器**にするなら、レリックと同じ `hooks:[{trigger, effect, params}]` を足すだけ
   （`EFFECTS` レジストリを武器も共有。例 `{ "trigger":"onAttack","effect":"applyPoison","params":{"amount":3} }`）。
   新しい効果が要るなら下記「レリック」と同じく `EFFECTS` に factory を1個追加（engine無編集）。
3. （任意）`web/src/ui/view.ts` で表示を追記。
→ engine.ts 不要。ただし**武器を1個増やすと開始武器の抽選分布が変わり全KPIが動く**ので
  `npm run sim -- --n 1000 --assert-kpi`（健全帯）→ `npm run gen-fixtures` → `npm run test`。

## レリック（効果）を足す
1. 効果が既存の `EFFECTS`（`web/src/data.ts`）で表現できるなら、`web/data.json` の `relics[]` に
   `{ id, name, archetype, desc, statMods, flags, hooks:[{trigger, effect, params}] }` を1件足すだけ。
2. **新しい効果**が要るなら `web/src/data.ts` の `EFFECTS` に factory を1個追加
   （`(params) => (g, owner, target, ctx) => { ... }`）。trigger は onAttack/onHitTaken/onTurnStart/onKill。
   - **ラン累計state（cap等のバランスノブ）**：factory は第2引数に `key`（= relic id）を受け取れる
     （`(params, key) => ...`）。`owner.effectState[key]`（Entity毎＝ラン毎にリセット）へ累計を貯めれば、
     上限/逓減を **data.json の params だけで調整可能**。例：報復の誓い `atkBuff` の `cap`（被弾累計+Nで頭打ち）。
     決定論を壊さないよう state は必ず `owner`/`g`（ラン毎に新規）に持つ（共有シングルトンの relic/closure に持たせない）。
→ engine.ts 不要。**doctor は件数をハードコードしない**ので、追加時に doctor 修正は不要（構造のみ検証）。

## 敵を足す
1. `web/data.json` の `enemies[]` に1件追加：`{ id, name, symbol, hp, atk, defense, behavior, sight, gold,
   tier, telegraph, spawnCap, params, note }`。`behavior` は既存のもの（melee/slow/erratic/ranged/support/boss/boss_twin）か新規。
2. 出現させるなら `spawnTable`（通常敵）か `run.bossPool`（ボス）に id を追加。
3. `web/src/ui/view.ts` の `ENEMY_ICON` にアイコンを1行追加。
→ behavior が既存なら engine.ts 不要（doctor は件数を自動カウント＝修正不要）。

## 敵AI（behavior）を足す ← レジストリ化済み
1. `web/src/behaviors.ts` の `BEHAVIORS` に関数を1個追加：
   ```ts
   BEHAVIORS.my_behavior = (g, e) => {
     const p = g.player;
     // g.adjacentToPlayer / g.stepToward / g.enemyAttackPlayer / g.lineClear /
     // g.enemiesNear / g.freeNear / g.occupied / g.spawnAt / g.msg と g.rng/level/state/enemies を使う
     // 調整値は e.etype.params.xxx ?? 既定値 で読む（data.json で調整可能に）
   };
   ```
2. その behavior を使う敵を `web/data.json` に追加（上記「敵を足す」）。
3. `doctor` が「全敵の behavior が登録済みか」を検証するので、未登録だと落ちて気づける。
→ **engine.ts の編集は不要**（`actEnemy` はレジストリへ委譲するだけ）。

## ボスを足す
1. 上記「敵AIを足す」でボス用 behavior を作る（勝利判定は `kill()` が `behavior.startsWith("boss")` を見るので
   **behavior 名を `boss` で始める**と自動でクリア扱いになる）。
2. `web/data.json` の `enemies[]` に追加し、`run.bossPool` に id を足す（ボス階でランダム出現）。
3. `ENEMY_ICON` にアイコン追加。

## 撃破報酬を足す（ボス等）← data駆動
1. 対象の `enemies[]` に `"reward": { gold?, heal?, relic? }` を足すだけ。倒した時に付与（`engine.grantKillReward`）。
   例：ボスに `{ "gold": 30 }`。`relic` は relic id（確定入手）。
2. ※現状ボスは最終フロアのみ＝撃破で勝利のため heal/relic は実質goldと同等（将来の**道中ボス**で heal/relic が活きる）。
→ engine.ts 不要。

## 初期装備を変える ← start ブロック
- `web/data.json` の `start` ブロック一箇所で管理：`{ hp, atk, weaponPool, relics }`。
  - `hp`/`atk`＝開始ステータス、`weaponPool`＝開始武器の抽選候補（順序維持）、`relics`＝開始所持レリックid配列。
  - 数値変更は data.json だけで反映（engine無編集）。`weaponPool` の増減は抽選分布が変わる＝KPIが動く点に注意。

## 数値（バランス）を調整する
- 敵AIの調整値 → その敵の `params`（例 boss の `slamChance` 等）。
- 全体ルール → `run`（回復率など）/ `tuning`（poisonAmp/poisonDecay/rewardChoices）。
- 変更後：`npm run sim -- --n 1000 --assert-kpi` で健全帯（35〜65%・5系統・停滞0）→ `npm run gen-fixtures` → `npm run test`。

## 状態異常を足す（やや高度）
- 付与は `EFFECTS`（applyPoison のような factory）で。減衰/集計は engine の `tickPoison` 付近に専用処理が要る
  （毒以外の状態異常はまだ汎用化されていない＝ここは engine 編集が必要な数少ない箇所）。

## 構造の地図（どこに何があるか）
- `web/data.json` … **数値の真実**（start/武器(+hooks)/レリック/敵(+reward)/spawn/bossPool/run/tuning/synergyCombos）
- `web/src/data.ts` … data.json ローダ＋**EFFECTS レジストリ**（レリック＋武器の効果を `bindHooks` で共有）＋`START` 等の型
- `web/src/behaviors.ts` … **BEHAVIORS レジストリ**（敵AI）
- `web/src/engine.ts` … 戦闘・ターン進行・報酬（フロア踏破＋`grantKillReward` 撃破報酬）・結果（`actEnemy` はレジストリへ委譲）
- `web/src/ui/` … 描画（view.ts）/テレメトリ（telemetry.ts）
- `web/scripts/doctor.ts` … 整合チェック（構造・behavior登録・tuning・start・CLAUDE.md参照／**件数はハードコードしない**）
- 詳細な開発ルールは [`../CLAUDE.md`](../CLAUDE.md)。
