# ローグライト MVP（Python / 標準ライブラリのみ）

設計仕様書（`../ローグライクゲーム_設計仕様書.md`）の MVP を実際に動く形にした実装。
依存ゼロ（Python 3.8+ の標準ライブラリのみ）で、PC でもこの環境でもそのまま動く。

## なぜ Python か（技術選定）
- **どこでも即実行・デバッグ容易**：インストール不要。`python3 -m roguelike ...` だけ。
- **統計が取れる**：Bot に数千ラン自動プレイさせ、仕様書 §12 の KPI を観測できる。
  ブラウザ実装ではこのバッチ検証が難しい。
- **拡張性**：敵・武器・レリックは `data.py` の「データ＋イベントフック」。
  コードを触らずバランス調整・要素追加ができる（§11 データ駆動）。
- **ビジュアルは最小・ゲーム性に集中**：端末の色付き記号で敵味方を即判別（§9）。

## 遊び方
```bash
# 対話プレイ（ttyならキー即応のcurses、非ttyなら自動で行入力にフォールバック）
python3 -m roguelike play --weapon sword

# 行入力モードを強制（どんな端末でも動く。w/a/s/d=移動・攻撃, .=待機, q=終了）
python3 -m roguelike play --line --weapon dagger

# Botの自動プレイを観賞（動いている様子の確認に最適）
python3 -m roguelike demo --build poison --weapon dagger

# KPI統計シミュレーション（面白さをデータで裏取り：§12）
python3 -m roguelike sim -n 1000

# 回帰テスト（生成の到達性・再現性・シナジー・停滞なしを検証）
python3 -m roguelike.tests
```

## 画面の見方
- `@`（緑）= あなた / `r s b a B h`（赤系）= 敵 / `D`（紫）= ボス / `>`（シアン）= 出口
- **赤背景のタイル = 予兆（危険）**：射手の射線・ボスのスラム範囲。ここから外れれば回避できる（§7 公平性）。
- 上部に HP バー・攻防・武器・遺物、下部に直近ログ（なぜそうなったかが分かる）。

## モジュール構成
| ファイル | 役割 |
|---------|------|
| `rng.py` | 乱数源のシステム別分離（再現性・デバッグ性 §11） |
| `data.py` | データ駆動の定義（武器・レリック・敵・出現テーブル） |
| `world.py` | 制約付きマップ生成＋到達可能性バリデーション（§4.1, §11） |
| `engine.py` | 戦闘・状態異常・敵AI（予兆付き）・ラン進行・KPI記録 |
| `bot.py` | 統計取得用ヒューリスティックAIプレイヤー |
| `ui.py` | 端末向け最小ビジュアル（render は純粋関数でテスト可能） |
| `simulate.py` | KPIレポート生成（§12） |
| `tests.py` | 軽量セルフテスト |

## 実装した設計の柱（仕様書との対応）
- **意味のある選択**：報酬は3択・トレードオフ（巨人の血=攻+6/HP-12 等）。
- **公平なランダム性**：生成後に経路探索で到達性を保証。遠隔/ボスは予兆で回避可能。
  治癒師など停滞要因の敵は1フロアの出現数を制約（§4.1）。
- **知識が報われる**：敵の行動パターン・効果は一貫。毒コンボなどシナジーは説明しすぎない。
- **ビルドの多様性**：火力 / 状態異常(毒) / 防御反射 / 持続 / 万能 の5系統が成立。
